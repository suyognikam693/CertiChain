# Decentralized Academic Credential Verifier
## Complete Setup Guide — Hardhat + Dual IPFS (Docker) + FastAPI

---

## Architecture Overview

```
┌──────────────────────────────────────────────────────────────────┐
│                         Your Machine                             │
│                                                                  │
│  ┌────────────┐    ┌──────────────────────────────────────────┐  │
│  │  FastAPI   │    │           Docker Compose                 │  │
│  │  :8000     │◄──►│  ┌─────────────┐   ┌─────────────────┐   │  │
│  │  (backend) │    │  │ ipfs-write  │   │   hardhat-node  │   │  │
│  └────────────┘    │  │ API  :5001  │   │   EVM RPC :8545 │   │  │
│        │           │  │ GW   :8080  │   └─────────────────┘   │  │
│  ┌────────────┐    │  └──────┬──────┘                         │  │
│  │  React     │    │         │ peered                         │  │
│  │  :3000     │    │  ┌──────▼──────┐                         │  │
│  │  (frontend)│    │  │  ipfs-read  │                         │  │
│  └────────────┘    │  │ API  :5002  │                         │  │
│                    │  │ GW   :8081  │                         │  │
│                    │  └─────────────┘                         │  │
│                    └──────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────────┘

Data flows:
  Issue:   FastAPI → IPFS write (pin)  → IPFS read (eager pin) → Hardhat (store hash)
  Verify:  FastAPI → Hardhat (check)   → IPFS read (fetch full credential)
```

## Step 2 — Install Blockchain Dependencies

```bash
cd credential-verifier/blockchain
npm install
```

This installs Hardhat and its plugins (~300MB, takes a minute).

Verify installation:
```bash
npx hardhat --version

```

## Step 3 — Start Docker Infrastructure (IPFS + Hardhat)

```bash
cd credential-verifier/docker

# Make init scripts executable (required on Linux/Mac)
chmod +x ipfs-write-config/001-configure.sh
chmod +x ipfs-read-config/001-configure.sh
chmod +x ipfs-read-config/002-peer-with-write.sh

# Start all services in background
docker compose up -d

# Watch logs to confirm startup (Ctrl+C to stop watching, containers keep running)
docker compose logs -f
```

## Environment Variables Reference

| Variable | How to Generate | Example |
|---|---|---|
| `CONTRACT_ADDRESS` | Output of `deploy.js` | `0x5FbDB231...` |
| `UNIVERSITY_ADDRESS` | Output of `deploy.js` (deployer wallet) | `0xf39Fd6e5...` |
| `UNIVERSITY_PRIVATE_KEY` | MetaMask export, or Hardhat account #0 for dev | `0xac0974be...` |
| `HARDHAT_NODE_URL` | Fixed: Hardhat Docker service URL | `http://localhost:8545` |
| `IPFS_WRITE_API` | Fixed: Docker service port | `http://localhost:5001` |
| `IPFS_READ_API` | Fixed: Docker service port | `http://localhost:5002` |
| `IPFS_WRITE_GATEWAY` | Fixed: Docker service port | `http://localhost:8080` |
| `IPFS_READ_GATEWAY` | Fixed: Docker service port | `http://localhost:8081` |
| `API_KEY` | `python -c "import secrets; print(secrets.token_hex(32))"` | `a3f9b2c1...` |
| `FRONTEND_URL` | Your React app URL | `http://localhost:3000` |

**Only 4 values need to be generated. The rest are fixed URLs.**

## Common Problems

### Docker containers won't start
```bash
docker compose down -v      # Remove volumes too
docker compose up -d        # Fresh start
docker compose logs -f      # Watch for errors
```

### "Contract not deployed" error
```bash
# Hardhat node restarted = blockchain reset = need to redeploy
cd blockchain
npx hardhat run scripts/deploy.js --network localhost
# Update CONTRACT_ADDRESS in backend/.env
```

### IPFS read node not peering with write node
```bash
# Manually trigger peering
docker exec ipfs-read sh /container-init.d/002-peer-with-write.sh

# Check peers
docker exec ipfs-write ipfs swarm peers
docker exec ipfs-read  ipfs swarm peers
```

### "Transaction reverted" on issue
```bash
# Usually means credential already exists (duplicate hash)
# Or: university account ran out of ETH (shouldn't happen on Hardhat)
docker exec hardhat-node npx hardhat accounts  # Check balances
```

### FastAPI can't reach Docker services
```bash
# If FastAPI runs locally (not in Docker): use localhost ports
HARDHAT_NODE_URL=http://localhost:8545   ✅
IPFS_WRITE_API=http://localhost:5001     ✅

# If FastAPI runs in Docker: use service hostnames
HARDHAT_NODE_URL=http://hardhat:8545     ✅
IPFS_WRITE_API=http://ipfs-write:5001   ✅
```

## Production Checklist

When moving from local Hardhat to a real network:

1. **Change `HARDHAT_NODE_URL`** → your cloud RPC endpoint (Alchemy, Infura, or your own node)
2. **Generate a real wallet** → MetaMask → export private key → fund with real ETH
3. **Redeploy contract** → `npx hardhat run scripts/deploy.js --network sepolia` (or mainnet)
4. **IPFS nodes** → run on a VPS with persistent storage; or migrate to Filecoin for guaranteed persistence
5. **`API_KEY`** → use a proper JWT system (python-jose) instead of a shared secret
6. **CORS** → restrict `allow_origins` to your actual frontend domain only
7. **HTTPS** → put FastAPI + IPFS gateways behind nginx with SSL
